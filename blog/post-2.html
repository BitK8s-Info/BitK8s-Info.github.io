Getting Started with Kubernetes 1.34

Kubernetes 1.34 introduces groundbreaking features including AI/ML workload optimizations, enhanced multi-cluster management, and revolutionary security improvements. Learn everything you need to know about this major release.

Kubernetes Tutorial DevOps Cloud Native
🚀

What's New in Kubernetes 1.34

Kubernetes 1.34 marks a pivotal moment in the evolution of container orchestration. This release brings over 60 enhancements with 22 features graduating to stable status. The focus is on AI/ML workloads, improved developer experience, and enterprise-grade security features that address modern cloud-native challenges.

✨ Headline Features

Native GPU scheduling, dynamic resource allocation for AI workloads, improved StatefulSet performance, and built-in secrets encryption are now stable features in Kubernetes 1.34.

Major Enhancements

1. Native AI/ML Workload Support

Kubernetes 1.34 introduces first-class support for AI and machine learning workloads. The new GPU resource management and scheduling capabilities make it easier than ever to run training jobs and inference workloads.

Here's how to request GPU resources in your pods:

apiVersion: v1
kind: Pod
metadata:
  name: ml-training-job
spec:
  containers:
  - name: pytorch-trainer
    image: pytorch/pytorch:latest
    resources:
      requests:
        nvidia.com/gpu: 2
        memory: "16Gi"
        cpu: "8"
      limits:
        nvidia.com/gpu: 2
        memory: "32Gi"
        cpu: "16"
  nodeSelector:
    accelerator: nvidia-tesla-v100

2. Dynamic Resource Allocation (DRA)

The new Dynamic Resource Allocation feature allows pods to request and release resources dynamically during runtime. This is particularly useful for batch processing and scientific computing workloads.

  • Runtime Flexibility: Adjust resource allocations without pod restarts
  • Better Utilization: Improved cluster resource efficiency by 40%
  • Cost Optimization: Scale resources based on actual workload demands
  • Priority Scheduling: Intelligent resource allocation for critical workloads

3. Multi-Cluster Management Improvements

Kubernetes 1.34 enhances multi-cluster operations with improved federation capabilities and cross-cluster service discovery. Managing applications across multiple clusters has never been easier. 

apiVersion: multicluster.x-k8s.io/v1alpha1
kind: ServiceExport
metadata:
  name: my-service
  namespace: production
spec:
  type: ClusterSetIP
  ports:
  - port: 8080
    protocol: TCP
  clusters:
  - us-west-1
  - us-east-1
  - eu-central-1

💡 Pro Tip

Use the new multi-cluster service mesh integration to automatically handle cross-cluster traffic routing and load balancing without manual configuration.

4. Enhanced Security Features

Security takes center stage in Kubernetes 1.34 with automatic secrets encryption, improved Pod Security Standards, and integration with external secret management systems.

Key Security Improvements:

  1. Automatic Secrets Encryption: All secrets are now encrypted at rest by default
  2. SPIFFE/SPIRE Integration: Native support for workload identity federation
  3. Network Policy v2: More granular control over pod-to-pod communication
  4. Runtime Security: Built-in container runtime security scanning

⚠️ Breaking Changes

Kubernetes 1.34 removes several deprecated APIs including batch/v1beta1 CronJob. Ensure you migrate to batch/v1 before upgrading. The legacy service account token mechanism is also removed in favor of bound service account tokens.

Getting Started with Kubernetes 1.34

System Requirements

Before installing Kubernetes 1.34, verify your environment meets these updated requirements:

  • Linux kernel 5.4 or higher (5.15+ recommended)
  • containerd 1.7+ or CRI-O 1.28+
  • At least 4GB RAM per control plane node
  • At least 2GB RAM per worker node
  • IPv4/IPv6 dual-stack support enabled

Installation with kubeadm

Installing Kubernetes 1.34 is streamlined with the latest kubeadm:

# Add Kubernetes repository
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.34/deb/Release.key | \
    sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] \
    https://pkgs.k8s.io/core:/stable:/v1.34/deb/ /' | \
    sudo tee /etc/apt/sources.list.d/kubernetes.list

# Install packages
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

# Initialize cluster
sudo kubeadm init \
  --kubernetes-version=v1.34.0 \
  --pod-network-cidr=10.244.0.0/16 \
  --service-cidr=10.96.0.0/12

# Setup kubectl
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

Upgrading from 1.33

Kubernetes 1.34 supports in-place upgrades from version 1.33. Follow this upgrade path: 

# Upgrade control plane
sudo kubeadm upgrade plan
sudo kubeadm upgrade apply v1.34.0

# Upgrade kubelet and kubectl on each node
sudo apt-get update
sudo apt-get install -y kubelet=1.34.0-* kubectl=1.34.0-*
sudo systemctl daemon-reload
sudo systemctl restart kubelet

⚠️ Upgrade Best Practices

Always upgrade your control plane first, then worker nodes. Test the upgrade in a non-production environment and ensure you have backups of etcd before proceeding.

Best Practices for Kubernetes 1.34

Resource Management

With dynamic resource allocation, adopt these new practices: 

apiVersion: v1
kind: Pod
metadata:
  name: adaptive-workload
spec:
  containers:
  - name: app
    image: myapp:latest
    resources:
      requests:
        memory: "512Mi"
        cpu: "500m"
      limits:
        memory: "2Gi"
        cpu: "2000m"
    # New: Resource resize policy
    resizePolicy:
    - resourceName: cpu
      restartPolicy: NotRequired
    - resourceName: memory
      restartPolicy: RestartContainer

Security Hardening

Leverage the new security features for a robust security posture: 

apiVersion: v1
kind: Pod
metadata:
  name: secure-app
spec:
  securityContext:
    seccompProfile:
      type: RuntimeDefault
    runAsNonRoot: true
    runAsUser: 1000
    fsGroup: 2000
  containers:
  - name: app
    image: myapp:latest
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop:
        - ALL

Migration Guide

Deprecated APIs Removed

The following APIs have been removed in 1.34:

  • batch/v1beta1 CronJob → Use batch/v1 CronJob
  • discovery.k8s.io/v1beta1 EndpointSlice → Use discovery.k8s.io/v1
  • node.k8s.io/v1beta1 RuntimeClass → Use node.k8s.io/v1

Feature Gate Changes

Several features have graduated and their feature gates are now enabled by default:

  • DynamicResourceAllocation (now stable)
  • PodSchedulingReadiness (now stable)
  • StatefulSetAutoDeletePVC (now stable)

Real-World Use Cases

AI/ML Training Pipeline

Deploy distributed machine learning training jobs with automatic GPU scheduling and resource scaling: 

apiVersion: batch/v1
kind: Job
metadata:
  name: distributed-training
spec:
  parallelism: 4
  template:
    spec:
      containers:
      - name: trainer
        image: tensorflow/tensorflow:latest-gpu
        command: ["python", "train.py"]
        resources:
          requests:
            nvidia.com/gpu: 1
            memory: "8Gi"
          limits:
            nvidia.com/gpu: 1
            memory: "16Gi"
      restartPolicy: OnFailure
